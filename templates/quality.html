Data Sources


We used two kinds of data sets to answer the two questions we formed. In order to answer the question on "How prevalent is Heart Disease in USA?" we used fact sheets published by Center for Disease Control and Prevention (CDC) and Henry J. Kaiser Family Foundation State Health Facts Project (respectivley found here and here.) Various statistics about heart disease prevalance are regularly collected and maintained by CDC. Hence this dataset seemed to be right choice to talk about the prevalance of this disease.

In order to answer the question on "What may be the causes of heart disease?" and "What are my chances of being suffering from heart disease?" We used dataset published on UCI data repository. This dataset contained various characteristics of patients such as their cholestrol level, whether they had chest pain, age, sex, etc. along with whether they were diagnosed with a heart disease. Hence we used this dataset to provide insights on what may be the causes of heart diseases using the information provided for diagnosed patients.

The datasets used for exploration from CDC did not require any processing. The dataset collected from Henry J. Kaiser Family Foundation provides free and up-to-date health data for all 50 states. The State Health Facts database is comprised of multiple healhty indicators from public and private sources that includes the Kaiser Family Foundation reports, public websites, government surveys and reports, and private organizations. Both these datasets were very clean and coherent with no outliers. Hence we only discussed the UCI heart disease dataset in the following sections which have interesting data quality characteristics.

UCI Heart Disease Dataset

The dataset used in our project was aggregated from four different datasets collected from Hungary, Switzerland, and the US. Percentage of aggregated data obtained from each source is shown in the adjacent pie chart. The data itself was available in a text file with each row broken up across several lines. Using a parser script we wrote in python, we aggregated data from these sources and stored it in csv format.


After aggregation, we had 899 records with 76 attributes. However, after initial exploration, we got rid of 26 variables which contained the same value across all records of the dataset. These variables included ssn, name, etc. pertaining to individual characteristics of the corresponding patients, which were anonymized by adding dummy values in their place.

Data Transformation

The target labels of this data initially ranged from 0 to 4, where 0 implied no presence of heart disease. It was not clear what values 1 through represent from the data representation, but they all denoted the presence of some form of heart disease. Hence we transformed 1 through 4 values to 1, so that 0 implies no heart disease and 1 implies presence of heart disease. With this transformed dataset we first performed missing value analysis.

Missing Value Analysis

We performed missing value analysis on the dataset with 50 variables remaining. Many variables had a large proportion of missing values which need to be imputed before we use it for prediction. We discarded all the variables (17 variables in the dataset) that have more than 10% missing values proportion so that we don't induce too much bias into the original data through imputation.
We then carefully examined each of the remaining variables to determine if it made sense to impute values for these variables. While some variable definitions were not very clear from the data sources, other variables such as day of exercise, month of exercise, etc. were too specific to be imputed based on mean/mode/standard deviation values. Removing these variables finally resulted in a dataset with 18 features and a binary target variable which denoted whether the patient suffered from heart disease.

Missing Values Imputation

We analyzed the pattern of missing values in this dataset to verify if they are missing at random.
One pattern that we found is that values from variables denoting if calcium blocker, nitrates, beta blocker, digitalis, and diuretic were used during exercise were missing together for most part. Particularly, there were 50 rows for which these features along with Resting Blood Pressure, Resting Heart Rate, Maximum Heart Rate, Exercise Duration, and Exercise induced Angina were all missing together. These seemed to be bad sample data points since its very unlikely for a patient suspected for heart disease will almost always have these values recorded. We decided to remove these points from our dataset in order to avoid adding bias to the data through imputation.
We then imputed missing values for each individual variable, using the following criteria:
1. Continuous variables - using standard deviation since it introduced minimum bias in the data
2. Categorical variables - using most commonly observed value

Outlier Analysis

We analyzed 20 variables to test for outliers. To start the analysis we made a box plot for each variable and calculated a a range of the values. Then we researched and collected the normal ranges for the variables from the internet to get a sense of the acceptable values.We observed records that had 0 resting bp.As we can see from the box plots below, resting BP and resting heart rate seemed to have high number of outliers.It is highly unlikely that a person will have that value so we replaced the outlier with mean.Similarly for resting heart rate, some records had values less than 20 while the normal range allows for values between 40-120. Hence, once again we replaced the outliers with the mean.
